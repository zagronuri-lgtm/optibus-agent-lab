# Run Readiness Mode

## Purpose

Run Readiness Mode determines whether a map has enough evidence for a simulated
controlled run review. It does not execute a real run.

## Entry criteria

- Map Audit Mode has produced baseline facts and risks.
- Scenario YAML identifies the intended run type and comparison baseline.
- Operator has reviewed unresolved audit issues.

## Allowed actions

- Read visible readiness indicators.
- Validate that required baseline KPIs exist.
- Compare scenario assumptions to baseline facts.
- Prepare an approval request for simulated run review.

## Forbidden actions

- Real Run.
- Save, Apply, Publish, Delete, Export, Import, Duplicate, Create Version.
- Advancing to Controlled Run Mode without an approval token.

## Major steps

1. Capture pre-readiness screenshot.
2. Review required KPI availability.
3. Review configured scenario assumptions.
4. Identify unresolved risks.
5. Produce approval-gate summary.
6. Capture post-readiness screenshot.

## Approval gate handoff

The approval request must include:

- Current workflow state.
- Intended action name.
- Expected typed token.
- Facts supporting readiness.
- Assumptions still requiring human acceptance.
- Risks that remain unresolved.

## Exit criteria

- The workflow either stops for human review or advances to Approval Gate.
- Any approval token is typed by a human operator, not generated by the agent.
